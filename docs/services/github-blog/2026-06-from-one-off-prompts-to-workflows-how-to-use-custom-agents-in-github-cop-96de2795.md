GitHub 블로그의 게시물은 Copilot CLI에 추가된 'custom agents' 기능을 소개합니다. 제공된 요약에 따르면 이 기능은 단발성 터미널 프롬프트를 팀의 반복 가능한 워크플로로 전환하도록 설계되었습니다.
피드 기준으로는 custom agents가 사용자의 스택과 팀 워크플로를 이해해 반복·검토 가능한 프로세스로 바꿔준다고 합니다. 구체적 사용법과 설정 절차는 원문을 확인해야 합니다.

## 핵심 포인트

- Custom agents로 터미널 프롬프트를 반복 가능한 워크플로로 전환
- 스택 및 팀 워크플로 컨텍스트를 반영해 실행과 검토를 지원
- GitHub Copilot CLI의 기능 확장 — 상세 설정은 원문 확인 필요

## 읽어볼 만한 이유

개발팀은 반복 작업을 표준화하고 리뷰 가능한 자동화 프로세스로 전환할 수 있습니다. 피드 기준으로는 생산성·일관성 개선 여지가 있어 기술 독자에게 관심거리가 될 가능성이 높습니다.
