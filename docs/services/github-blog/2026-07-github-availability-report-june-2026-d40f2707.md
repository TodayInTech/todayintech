# GitHub availability report: June 2026

> GitHub Blog · 2026-07-08 · Company news

---

6월 GitHub 보고서는 아키텍처 전환과 가용성 작업에서 ‘진전과 학습’이 동시에 일어났음을 담백하게 정리한다. Azure로의 모놀리식 트래픽 이전은 중앙 미주(Central US)에서 45% 피크를 기록했으나, 5월 21일의 안정성 문제로 한 달가량 램프를 중단했고 6월 17일 재개 시에는 단계별로 검증하는 per-turnup 안정성 게이트를 도입해 속도보다 신뢰성을 택했다고 설명한다. Git 트래픽은 월초 30%에서 43%로 성장해 50% 목표를 달성하지 못했으며, vPoP 트래픽 경로와 SSH의 엣지 읽기/쓰기 분리 미비가 주요 제약으로 지적되어 당분간 45% 전후에서 정체될 것으로 예상했다.
동시에 핵심 서비스의 분리·오프로드 성과도 제시된다. 익명 PR 읽기는 새 서비스 pullsd가 100% 처리하고, reposd는 REST 읽기의 50%까지 램프했으나 Redis 용량 제약으로 일시 중단했다. 사용자 서비스는 피크에서 초당 약 50만 쿼리를 주 DB에서 오프로드했고, API 레이트리미팅 결정의 약 97%가 게이트웨이에서 처리되어 모놀리스 내부 경쟁을 줄였다. 클라이언트 사이드 DB 로드 셰딩(실교통 5%)과 상호검증(두 명 확인)·통합 감사 추적 도입은 운영 탄력성과 보안 절차의 의미 있는 개선으로 보인다.
보고서는 또한 6건의 성능 저하 사고를 날짜별로 상세히 다루며 기술적 원인과 대응을 적시한다. 6월 4일 Copilot 코드리뷰는 새 의존성의 런타임 불일치로 대량 실패가 발생했고, 의존성 고정·호환성 검사·빠른 실패 로직 도입으로 재발 방지 계획을 제시했다. 6월 8일에는 익명 트래픽 급증으로 비인증 요청에서 504가 다수 발생해 로드밸런서·애플리케이션 층의 차단과 자동 탐지 강화가 이뤄졌다. 그 밖에 인증 구성 오류(memcached), 상류 모델 제공자 문제(Opus 4.8), 설정 배포 오류로 인한 모델 가용성 저하, 하이퍼바이저 문제로 인한 백그라운드 작업 지연 등 각각 원인에 맞춘 수정·운영 절차 개선을 약속했다. 전반적으로 ‘가용성→용량→기능’ 우선 원칙을 재확인하며, 수치와 구체적 대응을 투명하게 공개한 점이 기술적 의사결정과 향후 운영 리스크 관점에서 의미가 있다.
