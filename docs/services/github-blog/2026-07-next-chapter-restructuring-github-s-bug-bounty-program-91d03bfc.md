# Next chapter: Restructuring GitHub’s bug bounty program

> GitHub Blog · 2026-07-22 · Security

---

GitHub은 버그 바운티 프로그램을 연구자 경험과 ‘신호’에 집중하는 방향으로 재구성한다고 발표했습니다. 그 배경으로는 접수량 증가와 연구자풀의 빠른 확대가 거론되며, 이에 대응해 ‘소음’은 줄이고 고품질 보고서에 보상을 집중하겠다는 전략적 전환을 택했습니다. 핵심적 변화로는 초청 기반의 영구 VIP 프로그램 신설이 있으며, VIP 연구자는 더 높은 보상과 빠른 응답, 보안 엔지니어링 팀과의 밀착 협업 혜택을 받습니다. VIP의 정량적 보상표(저위험 $1,000 / 중간 $7,500 / 높음 $20,000 / 치명적 $30,000+)와 자격 기준(치명적 1건, 높음 2건, 중간 4건, 저위험 7건 중 하나 충족)이 공개되며 “많이 제출한다고 더 받는 구조”에서 “더 잘 제출하면 더 받는 구조”로 인센티브를 전환합니다.
공개 프로그램에서는 넓은 범위의 보상 대신 고정(스태틱) 지급액을 도입해 예측 가능성을 높였습니다(저위험 $250 / 중간 $2,000 / 높음 $5,000 / 치명적 $10,000). 동시에 공개 채널의 저품질·AI 생성 보고서를 줄이기 위해 HackerOne 신호 요건을 도입하고, 신호 기준에 도달하지 않은 연구자에는 최대 네 건의 초기 제출 기회를 허용해 신규 진입 장벽을 완화하는 한편 프로그램의 작동성을 유지하겠다고 밝혔습니다. 기존 접수된 보고서는 소급 적용되어 보상 체계가 변경되기 전의 기준으로 처리되며, 변경은 2026년 7월 27일 제출분부터 적용됩니다. GitHub은 더 빠른 응답, 명확한 심각도 근거 제시, 컨퍼런스·커뮤니티 교류 확대 등을 병행 투자해 심층적 연구를 장려하고자 합니다.
